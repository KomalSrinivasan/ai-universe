## 💡 How to Use This Site

=== "For Beginners"

    **Just starting with AI development?**

    1. Start with the [Getting Started Guide](guides/getting-started.md)
    2. Browse [Foundation Models](categories/foundation-models.md) to choose a model
    3. Check [Quick Reference](guides/quick-reference.md) for your use case
    4. Follow the deployment guides in each category

=== "For Developers"

    **Looking for specific tools?**

    - Use the **search bar** (top right) to find tools quickly
    - Browse by **category** in the left sidebar
    - Check [Quick Reference](guides/quick-reference.md) for use-case-based recommendations
    - View [Contributors](contributors.md) to see community members

=== "For Contributors"

    **Want to contribute?**

    1. Star the [GitHub repository](https://github.com/KomalSrinivasan/ai-universe)
    2. Read the [Contributing Guide](CONTRIBUTING.md)
    3. Submit a PR with your additions
    4. Join our growing community!

## 🎓 Learning Paths

!!! tip "Recommended Learning Paths"

    === "Build a Chatbot (Weekend)"
        1. [Choose a model](categories/foundation-models.md) - Qwen 2.5 or Llama 4
        2. [Deploy locally](categories/deployment-serving.md) - Using Ollama
        3. [Add monitoring](categories/observability-monitoring.md) - Using Langfuse
        4. Test and iterate!

    === "Build RAG Application (1 Week)"
        1. [Set up vector DB](categories/vector-databases.md) - Chroma or Qdrant
        2. [Choose RAG framework](categories/rag-knowledge.md) - LlamaIndex
        3. [Add observability](categories/observability-monitoring.md) - Helicone
        4. Deploy and monitor

    === "Production AI System (1 Month)"
        1. [Infrastructure setup](categories/infrastructure.md) - vLLM + distributed serving
        2. [Vector DB at scale](categories/vector-databases.md) - Milvus
        3. [Full observability](categories/observability-monitoring.md) - Langfuse + OpenLIT
        4. [MLOps workflow](categories/production-ai.md) - Prefect or Ray
